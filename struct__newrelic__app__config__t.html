Detailed Description

Agent configuration used to describe application name, license key, as well as optional transaction tracer and datastore configuration.

See Also
newrelic_create_app_config().

Field Documentation

char _newrelic_app_config_t::app_name[255]

Specifies the name of the application to which data shall be reported.

newrelic_datastore_segment_config_t _newrelic_app_config_t::datastore_tracer

Optional. The datastore tracer configuration. By default, the configuration returned by newrelic_new_config() will enable datastore segments with instance_reporting and database_name_reporting set to true.

char _newrelic_app_config_t::license_key[255]

Specifies the New Relic license key to use.

char _newrelic_app_config_t::log_filename[512]

Optional. Specifies the file to be used for agent logs. If no filename is provided, no logging shall occur.

newrelic_loglevel_t _newrelic_app_config_t::log_level

Optional. Specifies the logfile's level of detail. There is little reason to change this from the default value except under the guidance of technical support.

Must be one of the following values: NEWRELIC_LOG_ERROR, NEWRELIC_LOG_WARNING NEWRELIC_LOG_INFO (default), NEWRELIC_LOG_DEBUG.

char _newrelic_app_config_t::redirect_collector[100]

Optional. Specifies the New Relic provided host. There is little reason to ever change this from the default.

newrelic_transaction_tracer_config_t _newrelic_app_config_t::transaction_tracer

Optional. The transaction tracer configuration. By default, the configuration returned by newrelic_new_config() will enable transaction traces, with the threshold set to NEWRELIC_THRESHOLD_IS_APDEX_FAILING.
